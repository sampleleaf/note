window.document

The window object also contains a really important property named document, which is an object and it's the entry point into the DOM loaded in the current window. We can use the document object to select and control elements of the web page.

window.document會return整個HTML的document-object，HTML的元素透過DOM被視為object，這些object稱為node(節點)，可用js操控

可以將DOM視為HTML標記的樹狀結構，其結構來自一個root-document-node

EventTarget.addEventListener(type, listener)

eventTarget ➔ 觸發事件的元素

type ➔ 事件名稱(eventName)

listener ➔ 事件處理程序(eventHandler) ➔ eventHandler(eventListner)

※eventHandler是function，eventListner是function參數

※function參數是eventObject，eventObject.target就是eventTarget

事件委託(Event Delegation)

故意將event-handler綁定到parent-element上，而不是原本想執行event的每個child-element上，這樣可以增加效能，減少內存占用

原理是利用event-flow(capture-phase➔target-phase(在這邊指的就是child-element)➔bubble-phase)。點擊child-element時，event會傳播到parent-element，由parent-element來進行事件處理， 但要記得添加condition判斷，來指定想要觸法事件的child-element，否則parent-element自己也會被event影響到，造成非期望的效果

treehouseCourse說是利用bubble-phase，但把addEventListner的option設定為true(capture-phase觸發)，仍然有效果，故個人認為是event-flow

document.method()，選擇元素

舊語法：getElementById("")，getElementsByTagName("")，getElementsByClassName("")

新語法：querySelector("")，querySelectorAll("")

只要return複數elements，通常是回傳HTMLCollection或NodeList，雖然不太一樣，但它們都是DOM的類陣列對象(array-like object)

HTMLCollection vs NodeList

HTMLCollection不能直接使用array.method()，NodeList可以

NodeList包含元素節點和文字節點，HTMLCollection只包含元素節點而已

當利用js增加一個新element時，不刷新頁面的情況，HTMLCollection會即時更新(動態資料結構)，而NodeList則不會偵測到這個新元素(大部分情況是靜態資料結構)

document.method()之創建元素。node.property之添加節點內容。element.method()之將DOM元素送進window.document中

createElement([tagName])會創建DOM元素，但不在window.document中，所以不會顯示在網頁上

textContent添加內容到指定元素節點(創建的元素並沒有自帶內容)

append()跟prepend()將元素送進指定element的child裡，前者是放last-child，後者是放first-child

array.method()

forEach(function(para){})

迭代array的element，每個element只調用1次


filter(function(para){})

輸出新array，array中的element是根據"statement的boolean"來決定是否return，一般來說會用condition判斷boolean。

如果statement非boolean，javaScript會強制轉換成boolean

如果return"非0數字""非空字串"等，會判斷為true，element會回傳

如果return"0""空字串""null""undefined"等，會判斷為false，element不回傳


map(function(para){})

輸出新array，array中的element數量不變，根據statement來改變element而已


reduce(function(accumulator, currentValue){},[initialValue])

輸出一個value

initialValue沒設置的情況下，accumulator為array[0]，currentValue會迭代array[0]之後的element

initialValue有設置的情況下，accumulator為initialValue，currentValue會迭代array的element

但假如accumulator被改變，其內容會更新，不會一直是初始值
