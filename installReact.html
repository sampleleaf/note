react-install

react install for beginner

npx create-react-app my-app安裝react到my-app資料夾中

安裝完成後，進入my-app，可以執行以下的指令

npm start啟動一個開發用的伺服器，開發者可以在這邊進行測試

npm run buildreact會將原始程式碼編譯、壓縮並打包，生成用於可以部署到web伺服器上的靜態檔案，會看到有一個叫build的資料夾。此時因為已經產生最後的應用包，如果之後有修改程式碼，必須重新再run build一次

npm test用於測試程式碼，通常是用在片段，例如說components

npm run eject使用這個指令後，應用程式將不再使用Create React App提供的預設配置，而是使用自定義的配置，這意味著開發者需要自行管理和維護配置，而且無法復原，這是永久性的，新手不用動這個

簡單解釋react的src目錄裡的檔案

App.js：在這個檔案中，App函數中return內的東西可以修改，來改變網頁的頁面呈現

App.css：App.js利用import '[file]'導入app.css。這個文件可以刪除，我們可以用自己的style.css

index.js：

  • 新手一般不會動到這個檔案，只需了解index.js會導入app.js，然後用ReactDOM.render將app.js渲染到HTML指定的節點上
  • HTML指定的節點，react一開始是設置在作為template的public/index.html中的div.id="root"

reportWebVitals.js：用於監測和報告網頁性能指標，index.js也會導入這個檔案

menifest.json：主要與用戶在瀏覽器中看到的應用程式外觀和行為有關(新手知道這個就可以)。用於 Progressive Web App (PWA) 的配置文件，描述了應用程式的名稱、圖示、顏色和啟動方式等信息

App.test.js：主要用於測試

react+bootstrap

npm install react-bootstrap bootstrap

想添加bootstrap的css效果，得去app.js添加這行import 'bootstrap/dist/css/bootstrap.min.css';

如果想添加react-bootstrap標記，app.js要添加這行import { 標記 } from 'react-bootstrap';

如果不想塞太多標記在app.js裡，可以開其他js檔，添加這行import { 標記 } from 'react-bootstrap';並且建立一個function，將標記塞在裡面，然後用export default [function]，將function導出，再從app.js用import [function] from [fileRoute]

react install by VITE

npm create vite@latest

我們會用npm run dev啟動開發用伺服器，其實這些指令可以在package.json中的scripts找到

簡單處理vite安裝react後在src的檔案

app.jsx這其實不是什麼重要的東西，在預設裡，他只是component而已。main.jsx用<App />將component渲染到網頁上，可刪可不刪，要在這邊修改也可，畢竟他就是component

app.css這就更不用說了，他只是app.jsx這個component的css而已，刪了也無妨

main.jsx這個是react的entry-point

index.cssmain.jsx會import這個css檔案
