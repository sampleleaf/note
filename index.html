Semantic Elements & Css tricks

<header>

通常包含 logo / 主標題 / 網站描述 / <nav>（<header>的導航欄不要到別人的網站）

<nav>

主要跟"連結"相關，不限於只能在<header>裡

<main>

主要內容，可以包含多個<section>

此元素每個頁面使用一次就好。不要放重複的內容(例如header/footer/aside已經有的內容)

<section>

將內容以主題作分段，通常帶有標題

<article>

通常是獨立的文章，含標題 / 內容等等

<aside>

內容應該要與主要內容間接相關，可以是廣告 / 文章 / 連結

<strong>跟<em>

強調用，取代以前的<b>跟<i>

<figure>

通常是獨立的內容，影片 / 圖片 / 備註等等

備註會放在<figcaption>裡面

<blockquote>

引用了其他來源的東西，包含書籍 / 文章 / 名言等等

引用的作者會放在<cite>裡面

<footer>

通常包含 作者資訊 / 版權資訊 / 聯絡資訊 / <nav>（<footer>的導航欄通常是到其他個人社群）

當內容沒有超出範圍，讓<footer>貼齊下方空間的方法

content in the <body> without <footer>{
    min-height: calc(100vh - <footer>height)
}

rwd，偵測手機、平板橫向模式的顯示

@media (orientation: portrait) {
    content...
    }

portrait是指手機等的橫向模式

利用flex輕鬆將navbar的內容標題分成左右邊的方法

.container{
    display: flex;
}
.item-1{
    margin-right: auto;
}

想從第幾個開始分開，就選擇第幾個item
