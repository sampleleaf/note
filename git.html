  • git init
    創建新的local-repo
  • git config

    git config [--global] user.name "[name]"
    設定git中的使用者名稱
    git config [--global] user.email "[email]"
    設定git中的使用者信箱
    git config --global core.editor "code --wait"
    將默認編輯器改為VScode
  • git status
    顯示當前跟current-HEAD的差異，且有提示引導下一步or上一步(復原)
  • git log
    顯示commit記錄
    git log --oneline
    顯示commit記錄，每個記錄只顯示1行
    git log -d [-num]
    顯示commit記錄，並對比提交的修改（有點像git diff），顯示最近的num個提交記錄差異
  • git reflog

    git reflog show
    顯示git追蹤的歷史記錄(有時效性)
  • git diff
    只能查看working tree跟最新commit的差異
    git diff --staged
    只能查看staging area跟最新commit的差異
    git diff HEAD [file]
    可以同時查看working tree和staging area跟最新commit的差異
    若只想要看特定的檔案，可以輸入filename
    git diff [HASH]..[HASH]
    查看兩個存檔點的差異
    git diff [branch]..[branch]
    查看兩個分支的差異
  • git add

    git add [file]
    將指定檔案移至"準備提交區(staging area)"
    git add .
    將所有檔案移至"準備提交區(staging area)"
  • git commit
    提交至local-repo，提交時會開啟默認編輯器，可以方便輸入多行commit-log
    git commit -m "[message]"
    提交至local-repo，提交時可以順便輸入單行commit-log
  • git branch
    顯示所有local-repo的branch
    git branch [branch-name]
    創建分支，但不會切換分支
    git branch -d [branch-name]
    刪除分支(需要merge後才能刪除，想強制刪除改成-D)
    git branch -m [branch name]
    變更分支名稱(想強制更名改成-M)
    git branch -r
    查看所有remote-repo的branch
  • git switch

    git switch [branch-name]
    切換到指定分支
    git switch -c [branch-name]
    創建並切換到指定分支
  • git merge

    git merge [branch-name]
    當前分支合併指定分支
    情況3種
    第1種：只有指定分支有新進度(不打開編輯器，直接進行快速合併)
    第2種：兩個分支都有新進度，但檔案無衝突(會打開編輯器，關閉視窗則自動提交)
    第3種：兩個分支都有新進度，且檔案有衝突(打開編輯器，手動編輯衝突資料，且要手動提交)
  • git checkout

    git checkout [commit-hash||ref]
    HEAD追蹤hash或ref，回到之前的提交記錄/歷史記錄(想重新編輯只要創建分支即可)
  • git restore

    git restore [file]
    將指定的檔案還原到最近commit的樣貌(刪除也可以還原)
    git restore --source [commit-hash] [file]
    將指定檔案還原至指定commit的樣貌
  • git revert

    git revert [commit-hash]
    創建一個commit，這個commit是剛剛輸入的commit-hash紀錄(默認編輯器會跳出來讓我們編輯logname，關閉後即為提交)
    通常用於協作開發(刪除他人記錄並不禮貌)
  • git reset

    git reset [commit-hash||ref]
    移除commit-hash之後的所有提交記錄(歷史記錄不會被移除)，HEAD導回commit-hash或ref，但working-tree不會還原
    git reset --hard [commit-hash||ref]
    移除commit-hash之後的所有提交記錄(歷史記錄不會被移除)，HEAD導回commit-hash或ref，且working-tree會還原
  • git clone

    git clone [url]
    複製remote-repo到local
  • git remote
    顯示與local-repo相關聯的"remote-repo名稱"
    git remote -v
    跟git remote一樣，只是顯示更詳細的資訊
    git remote add [remote-name] [remote-url]
    使local-repo與remote-repo產生關聯
    git remote rename [old-name] [new-name]
    重新命名remote-repo
    git remote remove [remote-name]
    解除local-repo與指定remote-repo的關聯(並非移除remote-repo的資料)
  • git push
    local-repo資料上傳默認remote-repo的分支
    git push [-u] [remote] [branch]
    local-repo資料上傳指定remote-repo的分支，-u是設定默認上傳路徑
  • git fetch
    默認獲取origin-remote-repo的所有分支記錄
    git fetch [remote-name] [remote-branch]
    獲取remote-repo的提交記錄，但不影響local的提交記錄
  • git pull
    相當於git fetch加git merge
    默認獲取origin上跟local-branch同名的remote-branch的記錄
    git pull [remote-name] [remote-branch]
    跟git pull一樣，只是有指定remote的name跟branch
  • git rm

    git rm [file]
    刪除local-repo中的檔案（跟terminal的rm相比，只差在直接到staging-area而已，其實提交後效果都一樣）
  • git mv

    git mv [origin-filename] [new-filename]
    檔案更名（跟terminal的mv相比，只差在直接到staging-area而已，其實提交後效果都一樣）
